# RemoveBG Pro 两套具体方案解决方案

---

## 🎯 方案 A：免费额度+历史记录型

### 📋 方案概述
- **未登录用户：** 每天 3 次免费
- **登录用户：** 每月 50 次 + 历史记录

---

### 🏗️ 数据库设计扩展

```sql
-- 用户使用配额表
CREATE TABLE IF NOT EXISTS user_quotas (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL UNIQUE,
  -- 月度配额
  monthly_quota INTEGER DEFAULT 50,
  monthly_used INTEGER DEFAULT 0,
  quota_reset_date INTEGER, -- 下次重置日期的时间戳
  -- 历史记录
  total_used INTEGER DEFAULT 0,
  created_at INTEGER DEFAULT (strftime('%s', 'now')),
  updated_at INTEGER DEFAULT (strftime('%s', 'now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 未登录用户使用记录（基于 IP/设备）
CREATE TABLE IF NOT EXISTS guest_usage (
  id TEXT PRIMARY KEY,
  identifier TEXT NOT NULL, -- IP 地址或设备 ID
  date TEXT NOT NULL, -- YYYY-MM-DD 格式
  count INTEGER DEFAULT 0,
  created_at INTEGER DEFAULT (strftime('%s', 'now')),
  updated_at INTEGER DEFAULT (strftime('%s', 'now')),
  UNIQUE(identifier, date)
);

-- 索引
CREATE INDEX IF NOT EXISTS user_quotas_userId_idx ON user_quotas(user_id);
CREATE INDEX IF NOT EXISTS guest_usage_identifier_date_idx ON guest_usage(identifier, date);
```

### 🔌 API 设计

```typescript
// GET /api/usage/check
// Response:
{
  isLoggedIn: boolean
  remaining: number
  total: number
  used: number

  nextResetIn?: string // "25 天"
}
```

#### 2. 记录使用

```typescript
// POST /api/usage/record
// Body (登录用户):
{
  type: 'processing'
  historyId?: string // 关联的历史记录 ID
}

// Body (未登录用户):
{
  type: 'processing'
  identifier: string // IP/设备 ID
}

// Response:
{
  success: boolean
  remaining: number
  total: number
}
```

## 📊 两套方案对比

| 维度 | 方案 A：额度限制型 | 方案 B：纯记录型 |
|------|------------------|----------------|
| **未登录用户** | 每天 3 次 | 无限使用 |
| **登录用户** | 每月 50 次 + 历史 | 无限使用 + 历史 |
| **用户增长动力** | 获得更多额度 | 保存历史记录 |
| **收入模式** | 额度付费升级 | 功能付费升级 |
| **开发复杂度** | 🟡 中等（需要配额系统） | 🟢 简单 |
| **用户体验** | 可能被额度困扰 | 爽快无限制 |
| **转化率预估** | 较高（额度压力） | 中等（功能吸引力） |
| **推荐场景** | 想快速变现、控制成本 | 想快速增长、建立用户池 |

## 🎯 实施建议

### 如果选择方案 A：
1. 先实现基础的配额系统
2. 从较低的额度开始（每天 3 次，每月 50 次）
3. 观察用户行为和转化率
4. 根据数据调整额度

### 如果选择方案 B：
1. 快速上线，重点在用户体验
2. 用无限免费吸引大量用户
3. 通过高级功能变现
4. 建立用户社区和生态

---

## 💡 我的推荐

**早期阶段推荐方案 B** - 无限免费，快速获取用户，建立产品粘性

**理由：**
1. 背景移除工具竞争激烈，免费无限是强吸引力
2. 先建立用户池，再考虑变现
3. 开发简单，可以快速上线
4. 历史记录功能本身就是很强的登录诱因

**等用户量起来后，可以：**
- 保留方案 B 的无限免费
- 通过 Pro 会员的高级功能变现
- 或者过渡到混合模式
