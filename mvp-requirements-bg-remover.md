# MVP 需求文档：AI 图片背景去除工具

> 版本：v1.0 | 日期：2026-03-17

---

## 1. 项目概述

### 1.1 产品简介

| 项目 | 内容 |
|------|------|
| **产品名称** | RemoveBG Pro (图片背景去除工具) |
| **产品类型** | Web 应用 (单页面网站) |
| **核心功能** | 上传图片 → AI 自动去除背景 → 下载结果 |
| **目标用户** | 电商店主、设计师、自媒体创作者、普通用户 |

### 1.2 核心价值主张

- **简单快捷**：无需安装软件，浏览器打开即用
- **AI 驱动**：利用最先进的语义分割模型，精准识别主体
- **免费体验**：提供免费额度，降低使用门槛

## 4. 详细功能说明

### 4.1 图片上传 (F1)

**功能描述**：用户上传需要处理的图片

**交互流程**：
1. 用户进入页面，看到上传区域
2. 拖拽图片到上传框，或点击选择文件
3. 系统校验图片格式（JPG/PNG/WEBP）和大小（≤10MB）
4. 显示图片缩略图

**校验规则**：
- 支持格式：image/jpeg, image/png, image/webp
- 文件大小：最大 10MB
- 图片尺寸：最大 4096x4096

**异常处理**：
- 格式不支持 → 提示"请上传 JPG/PNG/WEBP 格式图片"
- 文件过大 → 提示"图片大小不能超过 10MB"
- 上传失败 → 提示"上传失败，请重试"

---

### 4.2 AI 抠图 (F2)

**功能描述**：调用 AI API 去除图片背景

**技术方案**：
- API 提供商：Rem.bg API / Clipdrop API / Baseten
- 处理方式：后端接收图片 → 调用 API → 返回结果

**交互流程**：
1. 用户上传图片后，点击"开始处理"
2. 显示加载动画（"AI 正在抠图..."）
3. API 返回结果后，显示处理后的图片
4. 处理失败 → 提示错误信息

**处理时长**：预计 3-10 秒（视图片大小而定）

---

### 4.3 效果预览 (F3)

**功能描述**：对比查看原图和处理结果

**交互方式**：
- 默认：左右分栏对比（左侧原图，右侧结果）
- 可选：滑块拖动对比

**界面布局**：
```
┌─────────────────┬─────────────────┐
│                 │                 │
│    原图         │    处理结果     │
│                 │                 │
└─────────────────┴─────────────────┘
```

---

### 4.4 下载图片 (F4)

**功能描述**：下载处理后的图片

**下载选项**：
- 透明背景 PNG（默认）
- 自定义颜色背景 JPG

**交互流程**：
1. 用户点击"下载图片"按钮
2. 浏览器触发文件下载
3. 文件名格式：`removebg_[原文件名]_[时间戳].png`

---

### 4.5 换背景色 (F5)

**功能描述**：为去除背景的图片添加新背景

**预设颜色**：
- 白色 #FFFFFF
- 蓝色 #0066CC（证件照常用）
- 红色 #FF0000（证件照常用）
- 透明（无背景）

**自定义颜色**：
- 取色器选择任意颜色

**交互流程**：
1. 用户点击颜色选项
2. 实时预览效果
3. 下载时应用选定颜色

---

### 4.6 处理进度 (F6)

**功能描述**：展示处理状态

**状态类型**：
| 状态 | 显示内容 |
|------|----------|
| 待上传 | "上传图片开始处理" |
| 上传中 | 进度条 + "上传中..." |
| 处理中 | 加载动画 + "AI 正在抠图..." |
| 完成 | 显示结果图片 |
| 失败 | 错误提示 + 重试按钮 |

## 6. 技术架构

### 6.1 系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   用户浏览器  │────▶│   后端服务   │────▶│  AI API    │
│  (前端页面)   │◀────│ (处理上传)   │◀────│(去除背景)  │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │
       │              ┌────┴────┐
       │              │ 图片存储 │
       │              │ (临时文件)│
       │              └─────────┘
       │
       ▼
┌─────────────────────────────────────┐
│          CDN (可选)                  │
└─────────────────────────────────────┘
```

### 6.2 技术选型

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| **前端** | React + Vite | 快速开发，响应式布局 |
| **UI 框架** | Tailwind CSS | 美观简洁 |
| **后端** | Node.js + Express | 轻量易用 |
| **图片处理** | Sharp | Node.js 图片处理 |
| **AI 服务** | Rem.bg API / Baseten | 抠图能力 |
| **部署** | Vercel (前端) + Railway (后端) | 简单快速 |

### 6.3 API 接入

**Rem.bg API 示例**：
```bash
curl -X POST https://api.rembg.com/v1/remove \
  -u user:api_key \
  -F image=@input.jpg \
  -o output.png
```

**Baseten (开源模型)**：
- 模型：U2-Net
- 费用：$0.002/张

## 7. 数据流设计

### 7.1 核心流程

```
1. 用户上传图片
      │
      ▼
2. 前端 → 后端：POST /api/upload (multipart/form-data)
      │
      ▼
3. 后端 → AI API：发送图片请求处理
      │
      ▼
4. AI API → 后端：返回处理结果 (PNG)
      │
      ▼
5. 后端 → 前端：返回结果 URL
      │
      ▼
6. 前端：展示处理结果，提供下载
```

### 7.2 接口设计

| 接口 | 方法 | 参数 | 返回 |
|------|------|------|------|
| `/api/upload` | POST | file (图片) | `{ resultUrl, thumbUrl }` |
| `/api/convert-color` | POST | imageUrl, color | `{ resultUrl }` |

## 8. 非功能需求

### 8.1 性能

| 指标 | 目标值 |
|------|--------|
| 页面加载时间 | ≤ 2 秒 |
| 图片处理时间 | ≤ 10 秒 |
| 并发处理能力 | 10 个请求/秒 |

### 8.2 可用性

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 99% |
| 图片处理成功率 | ≥ 95% |

### 8.3 安全

- 图片临时存储，自动清理（24小时内）
- API Key 存储在环境变量，不暴露在前端
- 限制文件大小和格式，防止恶意上传
